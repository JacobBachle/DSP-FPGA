# Contributing Guidelines

This repository follows a controlled collaboration model to ensure
technical rigor, documentation consistency, and research-quality output.

---

## 1. Access Policy

Direct write access is **not granted by default**.

All contributors must:
1. Submit the Contributor Intake Google Form
2. Receive approval from the maintainer(s)
3. Be assigned a unique contributor color
4. Be granted repository access

Unapproved pull requests may be closed without review.

## 3. Contributor Color System

Each contributor is assigned a **unique color**, used for:
- Learning resource descriptions
- Justifications for inclusion
- Documentation annotations

### Rules
- Use only your assigned color
- Use the same color consistently
- Do not change colors without approval

Color assignments are tracked by the maintainer.

## 5. Code Contributions

For RTL, testbenches, and scripts:
- Follow the existing directory structure
- Use clear commit messages
- Avoid committing generated files
- Comment non-obvious logic

Major architectural changes should be discussed before submission.

---

## 6. Review Process

All contributions are reviewed by the maintainer(s) and evaluated on:
- Technical correctness
- Documentation compliance
- Alignment with project goals

The maintainer reserves the right to request revisions or decline
contributions that do not fit the project scope.

---

## 7. Questions

For questions regarding access or contributions, refer to the project
documentation or contact the maintainer.
